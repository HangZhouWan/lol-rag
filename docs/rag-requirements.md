# 英雄联盟 RAG 系统 — 需求文档

> 版本: 1.0  
> 日期: 2026-06-22  
> 状态: 已确认

---

## 一、项目背景

已完成英雄联盟 Wiki 数据抓取（第一阶段），产出约 378 个 Markdown 文件，分为三大类：

| 类别 | 目录 | 数量 | 内容 |
|------|------|------|------|
| 英雄 (Heroes) | `data/heroes/` | ≤153 | 中英文名、称号、定位、背景故事、属性、技能（被动+Q/W/E/R） |
| 装备 (Equipment) | `data/equipment/` | ≤162 | 名称、图标、等级、价格、基础属性、主动/被动效果、合成路线、推荐英雄 |
| 符文 (Runes) | `data/runes/` | ≤63 | 名称、图标、类别、等级、效果描述 |

每个文件末尾包含来源 URL 和抓取时间元数据。

## 二、目标与定位

| 维度 | 决策 |
|------|------|
| 主要用户 | 普通玩家 — 游戏内快速查询英雄技能、装备属性、符文效果 |
| 次要目标 | RAG 技术学习与验证项目 |
| 问题类型 | 全面开放，不限制问题形式（事实查询、对比、推荐、推理均支持） |
| 交互语言 | 中文问答 |
| 回答速度 | 5-10 秒内可接受，优先保证答案质量 |
| 数据更新 | 静态快照，不自动更新，需要时手动重新抓取 |

## 三、交互界面

分两阶段推进：

### 3.1 第一阶段：CLI

- 终端启动后进入交互式对话循环
- 支持多轮上下文对话（记住最近 10 轮对话）
- `/clear` — 清空对话历史
- `/quit` 或 `Ctrl+C` — 退出程序
- 空白或无效输入给出友好提示

### 3.2 第二阶段：Web 聊天界面

**界面布局：**
- 左侧对话列表（新建 / 切换 / 删除对话）
- 右侧主聊天区域，对话气泡式展示
- 底部输入框，`Enter` 发送、`Shift+Enter` 换行

**对话管理：**
- 自动保存对话历史，刷新页面后恢复
- 每个对话独立上下文，支持多轮

**辅助功能：**
- 输入框上方提供快捷问题示例
- 支持导出对话记录为 Markdown
- 回答不满意可点击"重新生成"

## 四、回答质量要求

### 4.1 准确性

- 答案**必须基于**抓取的 Markdown 数据，不得编造不存在的信息
- 当数据不足以回答时，明确告知"当前数据中未找到相关信息"
- 事实类查询准确率应达到 95%+（基于测试集）

### 4.2 来源引用

采用**内联引用**格式：关键信息后标注 `[N]`，答案末尾列出引用列表。

示例：

> 冰霜之心的被动「凛冬之拥」可使附近敌人攻速降低 15% [1]，所受普攻伤害至多降低 40%，同时提供 20 技能急速 [1]。
>
> ---
> [1] 来源: data/equipment/冰霜之心.md

约束：
- 引用必须指向真实存在的文件
- 每条关键信息均可追溯来源

### 4.3 推理类问题

- 分析依据必须在数据中有迹可循
- 先列出分析依据，再给出结论
- 不要求推理绝对正确，但逻辑链必须透明

## 五、覆盖的问题类型

### 5.1 事实查询

- 英雄属性、技能详情、背景故事
- 装备属性、价格、效果、合成路线
- 符文类别、等级、效果

### 5.2 对比分析

- 多英雄属性/技能对比
- 多装备属性/性价比对比
- 多符文效果对比

### 5.3 推荐建议

- 英雄出装推荐
- 符文搭配推荐
- 英雄克制关系分析（基于数据推理）

### 5.4 模糊查询

- 用户描述不清时，系统列出可能的匹配项让用户确认
- 支持中文名/英文名/称号等多种方式识别英雄

### 5.5 交叉查询

- 支持跨类别查询（如"适合安妮的装备"需同时查英雄和装备数据）
- 三类数据之间的关联推理

## 六、错误与边界处理

| 场景 | 处理方式 |
|------|----------|
| 数据中无匹配信息 | 明确告知用户"当前数据中未找到相关信息"，不编造 |
| 输入过短（如单个字） | 提示用户补充更多信息 |
| 输入过长（超过 2000 字符） | 提示用户精简问题 |
| 对话历史超限 | 自动截断最早的消息，并提示用户 |
| 文件读取失败 | 记录错误日志，降级处理（跳过该文件继续回答） |
| 非法指令 | 提示可用指令列表 |

## 七、测试与评估

### 7.1 测试用例

- 准备 50+ 条测试用例，覆盖所有问题类型
- 包含正向用例、边界用例、模糊查询用例
- 预期分类分布：
  - 事实查询：~20 条
  - 对比分析：~10 条
  - 推荐建议：~10 条
  - 模糊查询 / 边界：~10 条

### 7.2 评估指标

- **事实准确率**：回答中事实信息的正确比例
- **引用正确率**：引用标注指向正确文件的比例
- **拒答正确率**：数据不足时正确拒答的比例

### 7.3 评估方式

- 提供评估脚本，批量运行测试用例并统计指标
- 支持人工标注对比

## 八、非功能性需求

- 代码结构清晰，模块职责单一
- 配置文件管理参数（模型选择、路径设置等）
- 日志记录关键操作（查询内容、耗时、是否命中）
- CLI 版本不依赖外部服务即可运行（除 LLM API 调用外）
